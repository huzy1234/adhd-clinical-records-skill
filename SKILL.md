# ADHD 临床病例管理系统

## 概述

本 Skill 用于自动化收集、管理 ADHD 儿科患者的临床信息。从病例图片中提取关键数据，生成飞书云文档，并填入飞书多维表格。

---

## 一、识别与提取规则

### 1. DSM-5 评分精准提取（踩点给分）

#### 注意缺陷型（9项）- 图片中勾选了几项就填几项
1. 经常难以保持注意力
2. 经常在任务活动中犯粗心大意的错误
3. 经常难以保持对任务的注意力
4. 经常难以按指令行事
5. 经常难以按计划完成任务
6. 经常回避、讨厌或勉强从事需要持续注意力的任务
7. 经常丢失学习用品
8. 经常被外界的刺激分心
9. 经常在日常活动中健忘

#### 多动-冲动型（9项）- 图片中勾选了几项就填几项
1. 经常坐立不安或敲打手脚
2. 经常在需要坐好的场合离开座位
3. 经常在不适当的场合跑动或攀爬
4. 经常难以安静地参与休闲活动
5. 经常"停不下来"
6. 经常说话过多
7. 经常在问题完成前就给出答案
8. 经常难以按顺序等待
9. 经常打断或插入他人活动

#### 亚型判断
- **ADHD-I（注意缺陷型）**：注意缺陷≥6项，多动-冲动＜6项
- **ADHD-HI（多动-冲动型）**：多动-冲动≥6项，注意缺陷＜6项
- **ADHD-C（混合型）**：两项都≥6项
- **待定**：两项都＜6项

### 2. 日期信息严格区分

**关键原则**：日期只记录本院（松江医院）就诊的日期，外院信息写入"既往史"

| 字段 | 说明 |
|------|------|
| 首诊日期 | 首次来本院就诊日期（不是外院确诊日期）|
| 首次评估日期 | 在本院首次做评估的日期 |
| 首次复诊日期 | 本院首次复诊日期（暂无则留空）|
| 中期评估日期 | 治疗中期在本院的评估日期 |
| 中期复诊日期 | 中期复诊随访日期 |
| 末次评估日期 | 最近一次在本院的评估日期 |
| 末次复诊日期 | 最近一次复诊日期 |
| 下次复诊日期 | 计划中的下次复诊日期 |

### 3. 既往史写入规则

外院确诊信息写入"既往史"字段，格式：
"外院确诊日期 + 医院名称 + 确诊疾病 + 后续治疗情况"

例如："2025-12-24上海儿童医学中心确诊ADHD，服药专注达3个月，阅读训练3个月"

## 二、工作流程

### 第1步：接收并分析图片
使用 `image` 工具分析用户上传的病例图片，提取文字信息。

### 第2步：创建飞书云文档
使用 `feishu_create_doc` 创建云文档，包含完整病例信息：
- 患者基本信息
- 7个时间节点日期
- 病史资料（主诉、现病史、个人史、既往史、家族史）
- DSM-5评分明细
- 量表评估结果
- 治疗方案

### 第3步：填入飞书多维表格

**表格信息**：
- 表格名称：ADHD 临床病例管理系统
- 表格链接：https://my.feishu.cn/base/EPIdbB1NiaJvxWsCxEpceh1enwy
- 表格Token：EPIdbB1NiaJvxWsCxEpceh1enwy
- 数据表ID：tbl1hHSDPBsDXodP

**字段顺序**（从左到右）：
1. 云文档链接（超链接格式：{"text":"链接标题","url":"https://..."}）
2. 姓名
3. 性别
4. 年龄
5. 首诊日期（毫秒时间戳）
6. 首次评估日期
7. 首次复诊日期（暂无则不填）
8. 中期评估日期
9. 中期复诊日期
10. 末次评估日期
11. 末次复诊日期
12. 下次复诊日期
13. 个人史
14. 既往史
15. 家族史
16. 病史摘要
17. 注意缺陷评分（格式"X/9"）
18. 多动冲动评分（格式"X/9"）
19. ADHD亚型
20. 药物治疗
21. 非药物治疗
22. 复诊计划
23. 确认（复选框）

**重要**：
- 每例病人必须填在**同一行**（新建记录）
- 日期用毫秒时间戳（如2025-12-24 = 1766534400000）
- 云文档链接必须是可验证的真实链接

### 第4步：更新Skill中的记录
在多维表格中查找是否已有该患者记录，如有则更新，无则新建。

## 六、注意事项

1. **评分必须精准**：根据图片中实际勾选的项目数填写，不是估计
2. **日期严格区分**：外院确诊日期写入既往史，本院日期才填入对应日期字段
3. **每例病人一行**：所有信息填入同一行，不要分多行
4. **云文档链接必须可打开**：使用完整的飞书文档链接
5. **隐私保护**：所有患者信息仅存储在用户飞书多维表格中
